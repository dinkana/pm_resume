## **Тестовое задание для менеджера проектов 2**

**Задание 1:**
Есть фрагмент переписки команды. Задача в Jira `PROJ-123` просрочена, следующий этап под угрозой. Какие данные вам нужны для анализа ситуации? Перечислите точные вопросы команде и владельцу задачи.

**Задание 2:** 
Нужно запустить функцию онлайн оплаты на сайте до 25 числа. В команде: 2 бэкенд-разработчика, 1 фронтенд-разработчик, 1 тестировщик. Известно: бэкенд-часть оценили в 10 человеко-дней, фронтенд — в 5, тестирование — в 3. Составьте примерный план работ (этапы, сроки, ответственные) на 15 рабочих дней, начиная с завтрашнего дня. Учитывайте риски и зависимости.

**Задание 3:**
Клиент в письме требует перенести срок сдачи этапа с 20-го на 15-е число из-за внутренних причин заказчика. Напишите текст краткого ответа клиенту. В ответе запросите необходимую информацию, обозначьте возможные последствия для проекта.

## Решение:

### Задание 1

**Данные для анализа:**
1. Критичность дедлайна для следующего этапа, есть ли резерв или последствия срыва?
2. Процент выполнения, какие работы остались?
3. Причины задержки, блокирующие факторы, технические сложности, переоценка объема?
4. Какие задачи или команды блокирует таску?
5. Доступность исполнителей, их текущая загрузка другими задачами?

**Вопросы команде и владельцу задачи:**

1. Какие конкретно шаги осталось выполнить для закрытия таски? Оцените оставшееся время в часах.
2. Что именно помешало выполнить задачу в срок?
3. Есть ли у вас сейчас все необходимое для завершения работы?
4. Назовите дату, когда вы гарантированно завершите задачу.
5. Можно ли декомпозировать задачу, чтобы часть функционала уже передать на следующий этап?

### Задание 2

**План работ:**

| Период (дни)  | Работы                                                     | Ответственный/Исполнитель                                  | Зависимости/Примечания                                                                 |
|---------------|------------------------------------------------------------|------------------------------------------------------------|----------------------------------------------------------------------------------------|
| 1-2           | Уточнение ТЗ, подготовка окружения                         | Менеджер проекта                                           | -                                                                                      |
| 1-10          | Параллельная разработка бэкенда                            | 2 разработчика (бэкенд)                                    | -                                                                                      |
| 3-10          | Разработка фронтенда                                       | 1 фронтенд-разработчик                                     | Подключается на 3-й день **после согласования API** с бэкендом                          |
| 5-12          | (Конкретный период) Разработка фронтенда                   | 1 разработчик (фронтенд)                                   | Зависит от готовности API бэкенда                                                      |
| 11-15         | Интеграционное и функциональное тестирование               | Тестировщик                                                | Наличие высокоприоритетных багов может сдвинуть релиз                                  |

**Риски и зависимости:**

1. Фронтенд не может начать полноценную работу без готового API бэкенда.
2. Неучтенные сложности в интеграции с платёжным шлюзом. Нужно выделить на эту работу первые 3 дня бэкенда для оценки.
3. Болезнь разработчика. Должен быть кросс-код ревью с первого дня + документация.
4. Фактическая разработка запланирована на 12 дней, оставлено 3 дня на непредвиденные сложности и финальное тестирование.

### Задание 3

**Текст ответа клиенту:**

> [!note]
> "Уважаемый [Имя клиента],
> 
> Получили ваш запрос о переносе сроков на 15-е.
> 
> Для оценки возможности переноса прошу предоставить:
> 1. Какие задачи из текущего этапа могут быть перемещены или упрощены для сокращения сроков?
> 2. Прошу рассмотреть увеличение бюджета для привлечения дополнительных ресурсов, чтобы мы смогли уложиться в указанный срок.
> 
> Обязательно проинформирую вас о влиянии данного переноса на
> - Качество тестирования последующих компонентов.
> - Сроки и стоимость следующих этапов проекта.
> 
> Жду ваших предложений по пунктам выше для оперативного анализа."
